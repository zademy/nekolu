# Code of Conduct

Nekolu is an open source project. We want the repository to stay useful, respectful, and safe for contributors, users, and maintainers.

## Our standard

When participating in this repository, please:

- be respectful, direct, and constructive
- assume good faith, but accept correction when needed
- keep technical discussions focused on facts, code, and reproducible behavior
- protect other people's privacy, credentials, and data
- use the project lawfully and responsibly

## Unacceptable behavior

The following behavior is not allowed in this repository or project spaces:

- harassment, hate speech, or personal attacks
- threats, intimidation, stalking, or doxxing
- trolling, baiting, or deliberately disruptive behavior
- spam, scams, or unrelated promotion
- publishing private data, secrets, access tokens, Telegram session files, or other sensitive material
- submitting code, issues, or documentation intended to enable malware, credential theft, privacy abuse, copyright infringement, or other unlawful activity
- bad-faith security reports or exploit publication that puts users at unnecessary risk

## Scope

This code of conduct applies to:

- issues
- pull requests
- code review discussions
- project documentation
- any repository-linked collaboration spaces

## Maintainer responsibilities

Maintainers may:

- edit, hide, lock, or remove content
- close issues or pull requests
- reject contributions
- limit or end participation in the repository

Moderation decisions are made to protect the project and its community.

## Reporting and enforcement

Repository-first reporting applies to this project.

- If the problem is in an issue, pull request, or comment thread, report it there when safe to do so.
- If there is no suitable thread, open a repository issue with the minimum details needed for maintainers to act.
- Do not include secrets, credentials, private chat content, or personal data in public reports.
- For platform abuse such as impersonation, doxxing, or threats, use GitHub's built-in reporting tools in addition to notifying maintainers through the repository.

## Responsible use

Nekolu is provided for lawful, authorized use. Users and contributors are responsible for how they operate the software, handle Telegram data, and comply with applicable law, platform terms, and third-party rights.

## Consequences

Depending on severity, maintainers may issue a warning, remove content, close contributions, temporarily restrict participation, or permanently ban a contributor from the repository.
